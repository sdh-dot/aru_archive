# 작업 마법사 (Workflow Wizard)

Aru Archive의 작업 마법사는 폴더 설정부터 분류 실행, Undo 확인까지를 9단계로 안내하는 순차형 워크플로우입니다.

## 진입

툴바의 **[작업 마법사]** 버튼을 클릭합니다.

## 9단계 구조

### Step 1 - 작업 폴더

- 분류 대상 폴더, 분류 완료 폴더, 관리 폴더, 앱 데이터 폴더 상태를 보여줍니다.
- **[작업 폴더 설정]** 버튼으로 분류 대상 폴더를 선택합니다.
- 선택한 폴더는 이름 변경 없이 그대로 Inbox 역할로 사용합니다.
- 같은 부모 폴더 아래에 `Classified`, `Managed` 폴더가 자동 생성됩니다.
- 앱 내부 데이터는 `C:\Users\<사용자명>\AruArchive` 아래에 저장됩니다.

### Step 2 - Scan / Load

- 현재 `Inbox` 경로와 파일 수, DB 그룹 수와 파일 수를 표시합니다.
- **[🔍 Inbox 스캔 실행]** 버튼으로 스캔을 시작합니다.
- 스캔은 배경 스레드에서 실행되며 완료 후 결과가 표시됩니다.

### Step 3 - 메타데이터 확인

- 메타데이터 상태 요약과 경고를 보여줍니다.
- 완전 중복 검사와 시각적 중복 검사를 실행할 수 있습니다.
- 중복 검사의 기본 범위는 `Inbox / Managed` 이며 `Classified` 복사본은 기본 제외됩니다.

### Step 4 - 메타데이터 보강

- `metadata_missing` 상태 중 Pixiv 식별자가 있는 파일을 대상으로 보강을 실행합니다.

### Step 5 - 사전 / 태그 정리

- alias, localization, 후보 태그, 외부 사전 상태를 확인합니다.
- 후보 태그 검토와 사전 가져오기/내보내기를 진행합니다.

### Step 6 - 태그 재분류

- 사전 변경 사항을 기존 그룹에 다시 반영합니다.

### Step 7 - 분류 미리보기

- 분류 대상 수, 예상 복사본 수, 예상 용량, fallback, 충돌 수를 보여줍니다.
- 위험도를 함께 계산해 보여줍니다.

### Step 8 - 분류 실행

- 분류는 원본을 이동하거나 삭제하지 않습니다.
- 분류 결과 복사본만 `Classified` 폴더에 생성합니다.

### Step 9 - 결과 / Undo

- 최근 작업 이력과 Undo 가능 상태를 확인합니다.
- `Classified` 폴더 열기와 작업 로그 진입을 지원합니다.

## 첫 실행 안내

처음 실행하면 작업 폴더 설정 화면이 먼저 열립니다.

- 사용자가 고른 폴더는 분류 대상 폴더로 그대로 사용됩니다.
- 같은 부모 폴더 아래에 `Classified`, `Managed`가 자동 생성됩니다.
- 앱 내부 데이터는 `C:\Users\<사용자명>\AruArchive`에 저장됩니다.

예:

- 선택 폴더: `D:\PixivInbox`
- 분류 대상 폴더: `D:\PixivInbox`
- 분류 완료 폴더: `D:\Classified`
- 관리 폴더: `D:\Managed`

## 권장 흐름

1. 작업 폴더 설정
2. Inbox 스캔
3. 메타데이터 상태 확인
4. 필요 시 중복 검사
5. 필요 시 메타데이터 보강
6. 사전 / 태그 정리
7. 태그 재분류
8. 분류 미리보기
9. 분류 실행
10. 결과 / Undo 확인

## 중복 검사 기본 범위

Step 3의 중복 검사 버튼은 기본적으로 `Inbox / Managed` 파일만 검사합니다.

| 검사 | 기본 범위 | Classified 포함 |
|------|-----------|-----------------|
| 완전 중복 검사 | Inbox / Managed | 제외 |
| 시각적 중복 검사 | Inbox / Managed | 제외 |

Classified 폴더의 복사본은 기본 제외됩니다. 분류 결과물이 원본과 중복으로 오탐되는 것을 막기 위한 정책입니다.
